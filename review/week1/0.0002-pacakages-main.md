package myutils

// Reverse returns the string reversed
func Reverse(s string) string {
    runes := []rune(s)
    for i, j := 0, len(runes)-1; i < j; i, j = i+1, j-1 {
        runes[i], runes[j] = runes[j], runes[i]
    }
    return string(runes)
}

#### 2. **Understanding `go.mod`**

Check your `go.mod` file - it should look like:

```go
module yourmodule

go 1.21  // or your Go version

require rsc.io/quote/v4 v4.0.1
```

#### 3. **Exported vs Unexported**

In Go:

- **Capitalized** = Exported (public): `quote.Go()`
- **Lowercase** = Unexported (private): `quote.internal()`

This is why `quote.Go()` works but `quote.go()` wouldn't.

#### 4. **Package Commands**

```bash
# Initialize a new module
go mod init yourmodule

# Add missing packages and remove unused ones
go mod tidy

# Download dependencies
go get rsc.io/quote/v4

# View all dependencies
go list -m all

# Update a package
go get -u rsc.io/quote/v4
```

### ğŸ“¦ Package Best Practices

1. **One package per directory**: All files in a directory should have the same package name
2. **Package naming**: Short, lowercase, single-word names (e.g., `http`, `json`, `fmt`)
3. **Avoid generic names**: Don't name packages `util`, `common`, `base` - be specific
4. **Doc comments**: Add a comment before package declaration:
   ```go
   // Package main demonstrates Go package usage.
   package main
   ```
